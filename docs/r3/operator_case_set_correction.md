# Исправление case-set R3, revision 2

Дата: 2026-07-15

## Выявленное расхождение

В зафиксированном case-set revision 1 был указан номер `0338200003326000131`. В фактическом завершённом прогоне ЕИС и в исходных файлах извещения обнаружен номер `0338300003326000131`.

Это не ошибка маршрутизации пользовательского интерфейса: архив и metadata run имеют один и тот же номер `0338300003326000131`, а XML извещения и имена вложений содержат этот же идентификатор. Следовательно, причина — опечатка при фиксации case-set.

## Решение

В `docs/r3/operator_case_set.yaml` повышена `revision` до `2`; номер исправлен на `0338300003326000131`, а прежнее значение сохранено в `previous_procurement_number`. Описание предмета приведено к тексту официального поля извещения ЕИС.

Остальные девять кейсов не изменялись. Batch-01 не изменяется и сохраняется как исторический результат revision 1. Batch-02 должен ссылаться на revision 2.

## Доказательства

- Источник фактического run: `toa-run-20260715130335-52d0da` в локальном runtime-хранилище.
- В metadata run `procurement_id`: `0338300003326000131`.
- В XML извещения и вложениях присутствует тот же номер.
- Отдельный аудит соответствия expected/observed выполняется до упаковки batch-02.
