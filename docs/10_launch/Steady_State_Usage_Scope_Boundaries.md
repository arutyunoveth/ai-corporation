# Steady-State Usage Scope Boundaries

## Allowed

- internal multi-deal usage
- operator-assisted execution
- manual-control cadence
- repeated operational cycles under documented review discipline

## Not Allowed

- external commercialization
- autonomous behavior claims
- self-serve positioning
- AI/runtime phase opening
- opening `M-049/M-050`
- reframing `M-052..M-055` as fully implemented runtime

## Concurrency Boundary

- concurrency must stay within the documented operator workload norms
- cycle growth must remain explicitly reviewed by the phase owner and reviewer

## Visibility Boundary

- visibility is sufficient only with documented rebuild/review cadence
- helper surfaces remain compensating controls, not passive notification runtime
