# Limited Runtime Design Non-Goals

## Explicit Non-Goals

- implementing `M-049 Agent Registry`
- implementing `M-050 Prompt / Schema Library`
- implementing runtime versions of `M-052..M-055`
- adding new endpoints, models, migrations, or execution paths for deferred slots
- adding AI/LLM/agent runtime behavior
- adding autonomous or self-serve claims
- silently reclassifying supporting slots as complete runtime

## Why These Are Non-Goals

The phase exists to translate planning into bounded design artifacts while preserving registry honesty. Any actual runtime opening must be handled by a later explicitly approved implementation phase.
