## Non-Blocker Observations

### Medium-Severity Operational Friction

1. Submission/procedure supervision still spans multiple persisted contours.
   - Impact: operator orientation is slower than ideal.
   - Compensating control: runbook + explicit control-gate checklist.

2. Delivery/payment/claim supervision depends on active helper rebuild cadence.
   - Impact: operators must deliberately rebuild visibility helpers instead of waiting for passive alerts.
   - Compensating control: scheduled manual review cadence during controlled pilot.

### Low-Severity Observations

1. Workspace feed and action queue are clear only when upstream helper build order is followed.
2. Closure review remains complete but still requires the operator to understand canonical/helper coexistence.
3. Early procurement and supplier/commercial justification is readable, but still distributed across multiple persisted sets.
