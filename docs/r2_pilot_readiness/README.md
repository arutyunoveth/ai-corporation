## Результат аудита

Построчная сверка зафиксирована в [`docs/r2/R2_LINE_ITEM_AUDIT.csv`](../r2/R2_LINE_ITEM_AUDIT.csv): 64/64 строк исходных XML сопоставлены с 64/64 строками отчётов как `MATCH_NORMALIZED`; `MISSING_FROM_EXTRACTION`, `DUPLICATED_IN_EXTRACTION`, `INVENTED_IN_EXTRACTION`, неверных количества/единиц/цен нет.

Первоначальный общий дефект `service_catalog: 0` исправлен в общем XML-extractor без ветвлений по номеру закупки. После повторного прогона извлечено 1, 34, 20, 6 и 3 строки — ровно по числу бизнес-строк в исходных XML.

`needs_review` сохранён для всех кейсов. Для двух сложных кейсов он обоснован отсутствующим/неизвлечённым проектом договора; это не маскируется заменой на GO. Для остальных ручная проверка остаётся обязательной из-за неполного извлечения отдельных приложений, но критических потерь позиций не обнаружено.

Web/JSON/DOCX/PDF повторно экспортированы после исправления.

## Воспроизводимый запуск

1. Настроить read-only токен ЕИС в `.env.local`.
2. Найти номера через публичный 44-ФЗ поиск.
3. Вызвать `create_run_from_eis_docs_archive` для каждого номера с `download_archive=True` и `analyze_after_download=True`.
4. Экспортировать `export_demo_agent_report_docx` и `export_demo_agent_report_pdf`.
5. Выполнить ручной чек-лист из [`manual_review.md`](manual_review.md).

Токены, архивы и персональные данные в Git не добавляются.
