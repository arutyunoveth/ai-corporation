# Pilot package

## Сценарий демонстрации

1. Ввести реестровый номер закупки.
2. Показать источник ЕИС и manifest документов.
3. Запустить анализ и открыть Web-отчёт.
4. Показать НМЦК, позиции, обязательные документы, риски и evidence.
5. Скачать DOCX и PDF и сверить ключевые поля с Web.
6. Показать `needs_review` и явно передать финальное решение специалисту.

## Ограничения

- Только 44-ФЗ и read-only доступ к ЕИС.
- Архив может быть неполным, сканированным или ещё формироваться.
- Автоматический анализ не заменяет юридическую и коммерческую проверку.
- GO без подтверждённых источниками цены, комплекта документов и условий запрещён.
- При отсутствии доказательства вывод должен быть `needs_review`, а не предположением.

## Вопросы пилотному пользователю

- Какие поля оператор проверяет в первую очередь?
- Какая доля позиций и документов должна быть подтверждена автоматически?
- Какие причины `needs_review` наиболее полезны?
- Какие выводы специалист чаще всего исправляет?
- Достаточна ли согласованность Web/DOCX/PDF для передачи заказчику?
- Какие действия продукт уже заменяет, а где человек должен оставаться в контуре?
