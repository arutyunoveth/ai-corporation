# Pilot Scope — Tender Agent

## Цель пилота

Проверить гипотезу: ИИ-агент сокращает время первичного отбора и анализа закупок для оператора тендерного отдела без снижения качества. По результатам пилота — дорожная карта расширения и коммерциализации.

## Срок пилота

2–4 недели с момента предоставления входных данных заказчиком.

## Периметр

- **Закон:** 44-ФЗ (публичные закупки).
- **Площадка:** zakupki.gov.ru (ЕИС).
- **Источник документации:** getDocsIP (read-only, токен физлица).
- **Тип анализа:** rule-based (детерминированный) — требования, вопросы, RFQ, ТКП, экономика, риски, рекомендация.
- **Роль агента:** первичный отбор и аналитическая подготовка.
- **Роль человека:** финальное решение, внешние действия (подача, подписание, отправка).

## Входные данные

- Профиль компании-поставщика (категории ОКПД2, регионы, ценовой диапазон, ключевые слова, стоп-слова, сертификаты, предпочтения по рискам).
- Список реестровых номеров закупок (или доступ к поиску на ЕИС).
- (Опционально) комплект документации по закупкам для сверки.
- (Опционально) XLSX файлы с коммерческими предложениями поставщиков.

## Роли сторон

### Arvectum

- Настройка и запуск агента под профиль заказчика.
- Адаптация ключевых слов и стоп-слов.
- Консультационная поддержка.
- Формирование отчёта по итогам пилота.

### Заказчик

- Предоставление профиля поставщика.
- Предоставление реестровых номеров закупок или доступ к поиску.
- Участие в оценке результатов.
- Обратная связь по качеству отчётов.

## Что делает Arvectum

- Разворачивает агента в локальной среде (on-premise или VPC по согласованию).
- Настраивает профиль поставщика на основе данных заказчика.
- Запускает обработку закупок по согласованному списку.
- Передаёт HTML-отчёты и журналы работы для оценки.
- Проводит интервью по результатам каждой итерации.

## Что делает заказчик

- Формирует и передаёт профиль поставщика.
- Передаёт реестровые номера или предоставляет доступ к ЕИС.
- Оценивает качество отчётов: полнота, точность, практическая применимость.
- Участвует в интервью и даёт обратную связь.

## Метрики успеха

| Метрика | Описание |
|---|---|
| Время первичного отбора закупки | От запроса до получения карточки с оценкой. Цель: < 1 мин. |
| Доля нерелевантных, отсеянных до анализа | % закупок, которые агент отбраковал по скорингу и оператор согласился. |
| Время получения и разбора документации | От нажатия кнопки до готового отчёта. Цель: < 5 мин. |
| Количество найденных рисков | Среднее количество рисков на одну закупку. |
| Количество вопросов к заказчику/поставщикам | Сформированных агентом. |
| Качество отчёта | Субъективная оценка оператора по 5-балльной шкале. |
| Доля пропущенных существенных условий | % условий закупки, не отражённых в отчёте (выборочная сверка). |

## Ограничения

- Пилот работает только с 44-ФЗ. 223-ФЗ и коммерческие закупки — вне периметра.
- Агент не подаёт заявки, не подписывает документы, не отправляет письма.
- Анализ rule-based; LLM-компонент не используется.
- Поиск на ЕИС — через публичный HTML (парсинг), не через API.
- При изменении структуры страниц ЕИС поиск переключается на ручной ввод номера.
- Для getDocsIP требуется токен, выпущенный на физическое лицо.
- Отсутствует автоматическое обновление статуса закупки (продление, отмена).

## Критерии завершения

- [ ] Обработано не менее 10 реальных закупок заказчика.
- [ ] Для каждой закупки сформирован HTML-отчёт.
- [ ] Проведена сверка отчётов по 3+ закупкам с полным ручным разбором.
- [ ] Собрана обратная связь от операторов заказчика.
- [ ] Зафиксированы расхождения и ошибки (если есть).
- [ ] Принято решение: переход к коммерческому контракту, расширение периметра или завершение.
