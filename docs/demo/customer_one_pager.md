# Arvectum Tender Agent — ИИ-агент для первичного отбора и анализа закупок

## Какая боль закрывается

- Оператор тратит 2–4 часа в день на поиск подходящих закупок и ручной разбор документации.
- Ключевые риски пропускаются из-за человеческой усталости при просмотре десятков страниц документации.
- Переключение между ЕИС, CRM, почтой и Excel замедляет принятие решения.

## Что агент делает

- Ищет закупки на ЕИС (44-ФЗ) по ключевым словам.
- Оценивает релевантность каждой закупки под профиль конкретного поставщика.
- Получает полный пакт документов из ЕИС через read-only сервис getDocsIP.
- Извлекает требования, вопросы к поставщикам, риски, готовит RFQ.
- Сравнивает коммерческие предложения (ТКП) и рассчитывает экономику.
- Формирует HTML-отчёт с рекомендацией для принятия решения.

## Как выглядит рабочий процесс

1. **Оператор** задаёт профиль поставщика (категории, регионы, цены, ключевые слова).
2. **Агент** выполняет поиск на ЕИС, оценивает релевантность, показывает карточки с баллами.
3. **Оператор** выбирает закупку и одним кликом запускает получение документации.
4. **Агент** через getDocsIP получает архив, распаковывает, анализирует документы.
5. **Агент** формирует отчёт: требования, риски, экономика, рекомендация.
6. **Оператор** проверяет отчёт, принимает финальное решение, выполняет внешние действия сам.

## Что уже реализовано в демо

- Консоль оператора с пятью вкладками: поиск, getDocsIP, загрузка, демо-набор, профиль.
- Публичный поиск 44-ФЗ с парсингом HTML и оценкой релевантности.
- Интеграция с getDocsIP для read-only получения документации по реестровому номеру.
- Pipeline анализа: требования → вопросы → RFQ → ТКП → экономика → риски → решение.
- Полный HTML-отчёт с разбивкой по блокам.
- Журнал работы агента для прозрачности каждого шага.
- Профиль поставщика с категориями, регионами, ценами, ключевыми словами.
- Детерминированный rule-based скоринг с цветными бейджами и детальной разбивкой.

## Какие данные нужны для пилота

- Профиль компании-поставщика (категории ОКПД2, регионы, ценовой диапазон, ключевые слова).
- 1–2 реестровых номера реальных закупок для теста getDocsIP.
- (Опционально) комплект документации по закупке для сравнения с результатом агента.
- (Опционально) примеры коммерческих предложений поставщиков в XLSX для теста сравнения ТКП.

## Что агент НЕ делает

- Не подаёт заявки на участие.
- Не использует ЭЦП.
- Не входит в личный кабинет ЕИС.
- Не обходит captcha.
- Не отправляет письма поставщикам.
- Не работает с 223-ФЗ (отдельный этап).
- Не использует LLM для анализа (отдельный этап, сейчас rule-based).
- Не сохраняет данные на внешних серверах.

## Безопасность и ограничения

- Токен ЕИС хранится только локально в `.env.local`, не коммитится в репозиторий.
- Архивы закупок распаковываются в локальную рабочую директорию.
- Все данные обрабатываются локально, без отправки на внешние API.
- Интерфейс не отображает полные URL архивов и ticket.
- В репозиторий попадают только обезличенные тестовые данные.
- Код готов к работе в изолированном контуре (on-premise / VPC).

## Формат пилота

- Срок: 2–4 недели.
- Участие Arvectum: настройка профиля, адаптация под процессы заказчика, консультации.
- Участие заказчика: предоставление профиля и данных, участие в оценке результатов.
- Результат: отчёт о качестве работы агента на реальных закупках заказчика.
- После пилота — дорожная карта расширения: 223-ФЗ, LLM, интеграция с CRM.

## Ожидаемый результат пилота

- Снижение времени на первичный отбор закупок в 2–3 раза.
- Прозрачная и воспроизводимая оценка релевантности.
- Снижение количества пропущенных рисков.
- Готовый отчёт для принятия решения по каждой закупке.
- Понимание, какие следующие функции наиболее критичны для заказчика.
