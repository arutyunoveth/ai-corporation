# Ручной прогон демо

Время: ~5–7 минут.

## Подготовка

- Открыть в браузере: `http://127.0.0.1:8001/demo/tender-agent`
- Перейти во вкладку **"Анализ закупки"**
- Использовать номер: `0323100010326000013`

---

## Шаг 1. Проверить готовность

**Действие:** ввести registry_number → нажать **"Проверить готовность"**.

**Что сказать заказчику:**
> "Сначала система проверяет, есть ли закупка в локальной базе, скачаны ли документы, извлечён ли текст, построены ли chunks и embeddings."

**Ожидаемый результат:**
- `ready_for_analysis=true`
- `chunks_total>0`
- `embeddings_total>0`

---

## Шаг 2. Подготовить закупку к анализу

**Действие:** нажать **"Подготовить закупку к анализу"**.

**Что сказать заказчику:**
> "Если закупка уже подготовлена, система не создаёт дубли, а показывает готовность по шагам. Если данных не хватает, запускаются нужные этапы подготовки."

**Ожидаемый результат:**
- `status=completed`
- steps показаны с иконками
- `ready_for_analysis=true`

---

## Шаг 3. Проанализировать закупку

**Действие:** нажать **"Проанализировать закупку"**.

**Что сказать заказчику:**
> "Теперь локальная LLM отвечает по индексированным документам, а система прикладывает источники. Это не ответ из головы, а анализ с привязкой к документации."

**Ожидаемый результат:**
- `status=completed`
- `sections_count=10`
- `sources_count>0`
- preview отчёта не пустой

---

## Шаг 4. Открыть отчёт

**Действие:** нажать **latest report / открыть отчёт**.

**Что сказать заказчику:**
> "Отчёт сохраняется и доступен повторно. Его можно использовать как черновик для дальнейшей проверки специалистом."

**Ожидаемый результат:**
- latest report открывается
- registry_number совпадает
- markdown/report отображается

---

## Шаг 5. Объяснить итог

**Короткий итог:**
> "Мы показали локальный контур: подготовка закупки, индексация документов, анализ локальной LLM и отчёт с источниками. Это основа агента, который помогает быстро разобрать закупочную документацию."
