# Ручной runbook демо

Цель: провести демо без знания внутренней архитектуры за 5–7 минут.

## Перед началом

- проверить backend health;
- проверить подключение к PostgreSQL и migration head;
- проверить embedding server `8090`;
- проверить LLM server `8088`;
- открыть demo URL.

## Demo URL

`http://127.0.0.1:8001/demo/tender-agent`

## Registry number

`0323100010326000013`

## Preset

`Mac mini: локальная LLM + Qwen3 embeddings`

## Mode

`fast / Быстрый`

## Шаги показа

### 1. Открыть вкладку анализа закупки

Показать, что это отдельный сценарий анализа закупки, а не общий чат.

### 2. Ввести registry_number

Использовать `0323100010326000013`.

### 3. Проверить готовность

Нажать `Проверить готовность`.

Что сказать:
> "Система сначала проверяет, готова ли закупка к анализу: есть ли документы, текст, chunks и embeddings."

### 4. При необходимости подготовить закупку

Если закупка ещё не подготовлена, нажать `Подготовить закупку к анализу`.

Что сказать:
> "Подготовка не делает лишних дублей. Если данные уже есть, система просто подтверждает готовность."

### 5. Запустить анализ

Убедиться, что выбран preset `Mac mini: локальная LLM + Qwen3 embeddings`, mode `fast`, затем нажать `Проанализировать закупку`.

Что сказать:
> "Анализ идёт локально, без облачной LLM, и выполняется как фоновая задача, поэтому интерфейс остаётся живым."

### 6. Показать progress по секциям

Во время выполнения показать:

- `job_id`;
- `status`;
- `progress_percent`;
- текущую секцию;
- что прогресс идёт по 10 разделам.

Что сказать:
> "Мы видим не просто спиннер, а понятный ход анализа по разделам документации."

### 7. Дождаться завершения

Ожидать `completed` или `completed_with_warnings`.

Что сказать, если анализ идёт долго:
> "Локальная LLM работает без облака, поэтому важнее смотреть на progress по секциям и не перезапускать анализ без причины."

### 8. Открыть отчёт

Нажать `Открыть отчёт`.

Показать:

- что отчёт структурирован;
- что в нём 10 разделов;
- что есть текст и источники.

### 9. Показать history

Прокрутить до history и показать, что запуск сохраняется по `analysis_run_id` и доступен повторно.

### 10. Скачать DOCX

Нажать `Скачать DOCX`.

Показать:

- файл скачался;
- файл открывается;
- внутри есть `registry_number`;
- внутри есть `analysis_run_id`;
- внутри есть текст отчёта.

### 11. Скачать PDF

Нажать `Скачать PDF`.

Показать:

- файл скачался;
- файл открывается;
- кириллица читается;
- внутри есть `registry_number`;
- внутри есть `analysis_run_id`.

## Приоритет источника данных

Приоритетным источником структурированных данных является электронное извещение (epNotification XML). Метаданные карточки поиска используются как fallback, а прикреплённые документы — для детального анализа. В случае конфликта данных приоритет имеет извещение ЕИС. Даты в мастере отображаются в формате ДД.ММ.ГГГГ, НМЦК — в рублях с разделителями тысяч. В отчётах источник данных маркируется как «электронное извещение ЕИС» или «карточка ЕИС».

## Что считать успешным результатом

- `status=completed` или `completed_with_warnings`;
- `sections_count=10`;
- `sources_count>0`;
- `analysis_run_id` присутствует;
- отчёт открывается;
- DOCX скачивается и открывается;
- PDF скачивается и открывается;
- кириллица в PDF читается.

## Если анализ идёт долго

- объяснить, что используется локальная LLM без облака;
- смотреть progress по разделам;
- использовать `fast` mode;
- не перезапускать анализ без причины.
