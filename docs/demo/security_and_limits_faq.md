# Security & Limitations FAQ — Tender Agent Demo

## Агент подаёт заявки?

Нет. Агент не выполняет никаких внешних действий. Он готовит аналитику и отчёт, а человек принимает финальное решение и выполняет подачу самостоятельно.

## Агент использует ЭЦП?

Нет. ЭЦП не используется и не будет добавлена в периметр демо/пилота. Подписание документов остаётся исключительно за человеком.

## Агент входит в личный кабинет ЕИС?

Нет. Агент работает только с публичными данными ЕИС:
- Публичный HTML-поиск (без авторизации).
- Сервис getDocsIP — read-only получение документации по номеру закупки.
- Нет входа в личный кабинет, нет сессий, нет cookies площадки.

## Агент обходит captcha?

Нет. Если ЕИС возвращает captcha или JS-heavy страницу, которая не парсится — поиск честно переключается на ручной режим: оператор открывает ЕИС в браузере, находит закупку и вставляет номер вручную. Агент не пытается обойти captcha.

## Где хранится токен?

Токен getDocsIP хранится в файле `.env.local` в корне проекта. Этот файл:
- Не коммитится в репозиторий (в `.gitignore`).
- Не отображается в интерфейсе.
- Не передаётся по сети за пределы работы SOAP-клиента.
- Доступен только локально на машине оператора.

## Что попадает в GitHub?

В репозиторий попадают:
- Исходный код.
- Тестовые данные (синтетические, обезличенные).
- Демо-профиль поставщика (обезличенный).
- Изолированные тестовые архивы (без реальных данных).

**Не попадают:**
- Токены и пароли.
- Реальные архивы закупок.
- Файлы из `.env.local`.
- Директория `company_agent_runs/`.
- Логи и временные файлы.

## Можно ли работать в закрытом контуре?

Да. Вся обработка данных — локальная. Агент может быть развёрнут без доступа к внешнему API (кроме ЕИС). Для полностью изолированной работы можно предоставить архив документации через ручную загрузку — поиск и getDocsIP не потребуются.

## Можно ли подключить локальную LLM?

Это отдельный этап. В текущем демо используется детерминированный rule-based анализ. Подключение LLM (локальной или через API) обсуждается после пилота, когда будут собраны метрики качества rule-based подхода и определены требования к глубине анализа.

## Что делает человек?

- Утверждает профиль поставщика и критерии релевантности.
- Выбирает закупки для анализа (из найденных агентом).
- Проверяет правильность извлечения требований.
- Загружает коммерческие предложения (если они не были получены автоматически).
- Проверяет расчёты экономики.
- Принимает финальное решение об участии в закупке.
- Выполняет внешние действия: подача заявки, подписание документов, отправка писем.

## Что происходит при ошибке ЕИС?

- Если getDocsIP не возвращает archiveUrl — run переводится в статус «нужна загрузка документов», оператор загружает файлы вручную.
- Если HTML-поиск не парсится — интерфейс показывает кнопку «Открыть поиск в ЕИС» и поле для ручного ввода номера.
- Если SOAP-сервис недоступен — ошибка логируется в диагностику, run не создаётся, оператор видит сообщение об ошибке.
- Если архив повреждён — извлекаются доступные файлы с предупреждением о неполноте.

## Можно ли использовать для 223-ФЗ?

Пока нет. 223-ФЗ требует отдельной интеграции, так как структура страниц и формат данных отличаются от 44-ФЗ. Мы планируем этот этап после завершения пилота на 44-ФЗ.

## Можно ли настроить профиль поставщика?

Да. Через вкладку «Профиль поставщика» можно задать:
- Категории продукции (ОКПД2 / ключевые слова).
- Регионы работы.
- Ценовой диапазон.
- Ключевые слова и стоп-слова для релевантности.
- Сертификаты и лицензии.
- Предпочтения по рискам (толерантность, макс. штраф, макс. задержка).

В текущем демо профиль хранится в памяти сессии. В пилоте профиль загружается из YAML/JSON файла.

## Можно ли подключить коммерческие предложения поставщиков?

Да. Агент поддерживает загрузку XLSX файлов с ТКП, нормализацию прайс-листов, сравнение предложений по позициям и расчёт экономики с учётом логистики и рисков. Это опциональная функция — загрузка доступна через вкладку «Загрузить документы».
